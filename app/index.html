GPExplorer


A Lisp based evolutionary computation research system.

GPExplorer has been designed using OOP and Lisp facilities for expressions evaluation. The focus is that experiment configurations can be buildt as "tasks" and then executed with some kind of scheduling, and adjust search algorithm parameters during the run.


Features General features
  • Objects for:
    • Search algorithm representation / coding
    • Object representation
    • Constants strategy (fixed / ephemeral)
    • Fitness evaluation
    • Other heuristics (CW savings heuristic, etc.)
  • Serialization for multi-image over a TCP connection for multiprocessing
  • GUI for Windows 7 or superior or background interface
  • Text reporting utility
EC features
  • NSGA-2 implementation (need test and results report)
  • Main evolutionary algorithm variants (generational, steady state)
GP Tree features
  • Implementation for ADF or SRT {REVIEW, TEST}
  • Grammatical evolution and a set of general tree operators
  • Examples (6/11-mux, regression, etc.)
Other features
  • Linear ordering using lists representation (example) {REVIEW, TEST}

Currently, site is divided in two parts, the first with graphical interfaces used for build some internal examples; and the other for GP live search examples on our server.
