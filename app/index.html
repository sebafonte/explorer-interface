GPExplorer


A Lisp based evolutionary computation research system

GPExplorer has been designed using OOP and Lisp facilities for expressions evaluation. Experimental configurations are represented as "tasks" and then executed with some kind of scheduling, maybe adjust search algorithm parameters during the run.


Features General features
  • Objects for:
    • Search algorithm objects for representing different EC configurations
    • Representing problems
    • Constants strategy (fixed / ephemeral)
    • Fitness evaluation
    • Other heuristics (CW savings heuristic, etc.)
  • Park Miller random numbers generator
  • Serialization for multi-image over a TCP connection for multiprocessing
  • GUI for Windows 7 or superior
  • Text reporting utility
  • Objects for multiobjective optimization methods
EC features
  • NSGA-2 implementation (need test and results report)
  • Main evolutionary algorithm variants (generational, steady state)
GP Tree features
  • Implementation for ADF or SRT {REVIEW, TEST}
  • Grammatical evolution and a set of general tree operators
  • Examples (6/11-mux, function regression, etc.)
Other features
  • Linear ordering using lists representation (example) {REVIEW, TEST}

Currently, site is divided in two parts, one with graphical interfaces used for build some internal examples; and the other for GP live search examples on our ad-hoc cloud server.
