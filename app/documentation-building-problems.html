Building problems

Building tasks

Using CAPI GUI problem building is done using configuration panes. When configured, the problem can be saved (as a tasks) or the entire environment including most available panes.

First we open a pane for building tasks and define the problem, language, algorithm, initial generation method, fitness-evaluation, etc. Some options get reset when an invalid combination has been detected, as well as filtering possible options. The task can be tested and then it can be evaluated using any other kind of planifications (using it from command interface with SO task scheduler by example).

Example use case video here:

  • Open search tasks pane
  • Select objective class (problem)
  • Select planifier
  • Create task and run
